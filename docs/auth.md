# 用户和游客会话设计草案

## 首版策略

- 首版要求登录后才能使用 Branchborn 作画能力。
- 未登录访问首页、项目库和 Canvas 页面时，跳转到登录页。
- 未登录调用 Branchborn 项目、上传、Agent、生图、任务和资源 API 时，返回 401。
- 登录用户可以创建项目、上传图片和执行生图。

## 游客会话

- 游客会话仅作为旧项目迁移来源保留。
- 登录后，如果当前浏览器存在旧 `branchborn_guest_session` Cookie，服务端会把该 Cookie 哈希对应的游客项目自动迁移到当前登录账号。
- 迁移完成后删除游客 Cookie。
- 无痕窗口或换浏览器没有原 Cookie 时，无法自动找回旧游客项目。

## 登录用户

- 登录能力沿用 Supabase Auth。
- 登录用户数据通过 `auth.uid()` 建立所有权关系。
- Branchborn 主路径使用 `projects.user_id` 作为项目、消息、资源、画布节点和任务的所有权边界。

## 风险控制

- 单个用户最多同时存在 `2` 个进行中的生图任务。
- 首版暂不限制每日生图次数。
- 如果后续出现滥用，再增加 IP 级限流、验证码或每日额度。
